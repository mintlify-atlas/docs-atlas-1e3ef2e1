---
title: "Quickstart"
description: "Enable NoodleMapper on a difficulty and add your first map range in under five minutes."
---

This guide takes you from a fresh NoodleMapper install to having a named, color-coded range sitting on your editor timeline. It assumes NoodleMapper is already [installed](/installation) and ChroMapper is running.

<Steps>
  <Step title="Open ChroMapper and navigate to a song">
    Launch ChroMapper. On the Song Select screen you can already see the **Clone from Git** button in the top-right of the song info panel — that's NoodleMapper running in the `SongSelectMenu` scene. Select any song you want to annotate and open it.
  </Step>

  <Step title="Open the Song Edit Menu">
    You are now in the Song Edit Menu (`SongEditMenu` scene). Look at the **bottom-right corner of the song info panel**. You will see two new buttons added by NoodleMapper:

    - **NoodleMapper** (grey) — opens the Map Config window for enabling NoodleMapper per difficulty
    - **Git** (blue-grey) — opens the Git version control window for this song

    Click **NoodleMapper**.
  </Step>

  <Step title="Enable NoodleMapper on a difficulty">
    The **Map Config** window lists every characteristic and difficulty combination for the song. Each row has an **Enable NM** toggle.

    Toggle on the difficulty you want to annotate, for example `ExpertPlusStandard`.

    <Note>
      Enabling NoodleMapper writes the key `_nmMap` into that difficulty's `CustomData` in `Info.dat`. The value is set to the canonical name of the difficulty (`ExpertPlusStandard` in this case). This is what tells NoodleMapper to load the matching `.map` file when you open that difficulty in the editor.
    </Note>

    Close the Map Config window. ChroMapper will show the difficulty row as unsaved (dirty). Save it using ChroMapper's normal save controls before opening the difficulty.
  </Step>

  <Step title="Open the difficulty in the mapper">
    Load the difficulty you just enabled in the mapper, the same way you normally would in ChroMapper. NoodleMapper's `EditorManager` scene manager activates and looks for the `.map` file referenced by `_nmMap` in `CustomData`. If no file exists yet, it starts with an empty `MapData` object.
  </Step>

  <Step title="Open the NoodleMapper editor window">
    Press **Ctrl+N**. The NoodleMapper editor window opens.

    If the difficulty does not have NoodleMapper enabled, the window shows the message `NM isn't enabled in this difficulty` instead of the editing controls. Go back to the Song Edit Menu and enable it first.

    The window has two sections:

    - **Active modmap** — a dropdown to select a `.modmap` file to attach to this difficulty (leave it on `-none-` for now)
    - **Map ranges** — the list of named beat ranges for this difficulty
  </Step>

  <Step title="Add a map range">
    In the map ranges section, click **new...**

    NoodleMapper adds a new range with:
    - An auto-generated name (`Range 1`, `Range 2`, etc.)
    - A randomly generated color
    - Start beat set to the current playhead position
    - End beat set one beat later

    You will see the range appear as a colored bar on the editor timeline immediately.

    To adjust the range:

    1. Move the playhead in ChroMapper to the beat where you want the section to start.
    2. Click the **=** button on the left side of the beat range display. This sets the start beat to the current playhead position.
    3. Move the playhead to the beat where the section ends.
    4. Click the **=** button on the right side. This sets the end beat.
    5. Edit the name in the text field to something meaningful, for example `chorus`.

    To change the color, click the colored square to the left of the name field. ChroMapper's color picker opens.

    <Tip>
      If you name two ranges identically (for example, two sections both called `verse`), NoodleMapper automatically copies the color from the first matching range. This keeps your color-coding consistent across repeated sections.
    </Tip>
  </Step>

  <Step title="Save your map">
    Save the map using ChroMapper's normal save shortcut or menu. NoodleMapper hooks into ChroMapper's `BaseDifficulty.Save` method and writes the range data automatically at the same time.

    The data is written to:
    ```
    YourSong/
    └── NoodleMapper/
        └── ExpertPlusStandard.map
    ```

    The `.map` file is JSON and looks like this for a single range:

    ```json
    {
      "ranges": [
        {
          "s": 32.0,
          "e": 64.0,
          "c": { "r": 0.8, "g": 0.4, "b": 1.0, "a": 1.0 },
          "n": "chorus"
        }
      ]
    }
    ```

    Fields: `s` = start beat, `e` = end beat, `c` = color (RGBA), `n` = name.
  </Step>
</Steps>
