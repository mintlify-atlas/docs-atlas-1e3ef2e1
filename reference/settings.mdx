title: Settings
description: Reference for NoodleMapper's settings system, including file location, API methods, and key path conventions.
---

NoodleMapper stores plugin-wide configuration in a single JSON file managed by the `Settings` static class. Settings persist across editor sessions and are read lazily on first access.

## File location

```bash
{Application.persistentDataPath}/NoodleMapper.json
```

`Application.persistentDataPath` is a Unity path that resolves to a platform-specific location, typically:

| Platform | Typical path |
|----------|------|
| Windows  | `%APPDATA%\..\LocalLow\BeatGames\ChroMapper\NoodleMapper.json` |
| macOS    | `~/Library/Application Support/BeatGames/ChroMapper/NoodleMapper.json` |
| Linux    | `~/.config/unity3d/BeatGames/ChroMapper/NoodleMapper.json` |

<Info>
  The settings file is created automatically on first access if it does not exist. You do not need to create it manually.
</Info>

## API reference

### Settings.Get

```csharp
public static JSONNode? Get(string name, JSONNode? defaultValue = null)
```

Reads a value from the settings file by key path. If the key is not present and `defaultValue` is provided, the default is written to the file and returned.

| Parameter | Type | Description |
|-----------|------|-------------|
| `name` | `string` | Dot-separated key path (e.g., `"editor.showRanges"`). |
| `defaultValue` | `JSONNode?` | Value to write and return if the key is missing. Optional. |

**Returns:** The stored `JSONNode`, or `null` if the key is absent and no default was supplied.

```csharp Example
// Returns the stored value, or writes and returns the default if not set
var value = Settings.Get("myKey", new JSONString("default"));
```

---

### Settings.Set

```csharp
public static JSONNode Set(string name, JSONNode value)
```

Writes a value to the settings file at the given key path. The file is saved to disk immediately after each call.

| Parameter | Type | Description |
|-----------|------|-------------|
| `name` | `string` | Dot-separated key path. |
| `value` | `JSONNode` | Value to store. |

**Returns:** The value that was written.

```csharp Example
Settings.Set("myKey", new JSONString("myValue"));
```

---

### Settings.Reload

```csharp
public static void Reload()
```

Discards the in-memory settings and re-reads the file from disk. Use this if the settings file has been edited externally.

```csharp Example
Settings.Reload();
```

## Key path conventions

Keys use **dot notation** to address nodes within the JSON object. The `Settings` class uses `Data.GetNode` and `Data.SetNode` internally to resolve paths.

```csharp Key path examples
// Reads/writes the top-level key "myKey"
Settings.Get("myKey");

// Reads/writes a nested path: { "section": { "item": ... } }
Settings.Get("section.item");
```

<Note>
  Key path traversal is handled by `Data.GetNode` and `Data.SetNode`, which split the key on `.` and walk the JSON object tree. A single-segment key addresses a top-level field; a multi-segment key creates or traverses nested objects.
</Note>

## Example settings file

A newly created settings file starts empty. Keys are only present if code has explicitly called `Settings.Set`.

```json NoodleMapper.json
{}
```

<Note>
  NoodleMapper does not define any built-in settings keys in the current version. The settings system is available for use by code that calls `Settings.Get` / `Settings.Set`.
</Note>

---

## Auto-save behavior

`Settings.Set` writes the full settings file to disk on every call. There is no explicit save step. If you need to make multiple changes efficiently, be aware that each `Set` call triggers an I/O write.
